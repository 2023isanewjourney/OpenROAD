Detailed Description

This program implements the core, a Nesterov nonlinear optimizer.

Compared to Conjugate Gradient methods (or a similar algo: gradient descent), Nesterov resolves the problem in an efficient way by looking a head term (eg. Iter steps, learning rate) in its equation.

int gpl::NesterovPlace::doNesterovPlace ( int  start_iter = 0)

runs Nesterov Placement.

A Nesterov’s method solves the nonlinear problem iteratively till convergence is reached. A few variables are maintained and updated while iteratively solving the optimization problem with params, steplength (α) - how far to move in a direction, which is maximized in order to accelerate the convergence based on a reference solution - Lipschitz constant. In addition, stopping criteria such as minimum overflow, divergence detection are specified at beginning.

Precondition
no divergence detected.
Postcondition
Parameters
start_iter
Returns
iteration count.

Core Nesterov Loop of ePlace-MS: It computes the gradient and preconditioner, predicts the Lipschitz constant, and adjusts steplength via backtracking.

  • Algo I: step 4 here, prevA is a_(k), curA is a_(k+1), where a_(k) is an optimization parameter iteratively updated. For further information, see the ePlace-MS paper's Algorithm 1.
  • Algo I: step 5 partially to compute the reference solution coeff is (a_k - 1) / ( a_(k+1) ) in paper.

(LOOP_START: BkTrk) Back-Tracking loop to effectively prevent steplength overestimation

  • Algo I: step 3 Compute a new solution given the computed gradient and steplength
  • Algo I: step 5 Computes a new reference solution which will be used to compute a reference step length at the following steps
  • Update next Coordi with the new computed nextCoordi using given steplength, without boundary exceeded. Otherwise, update by heuristic rules, such as bg_.lx() + gCell->dDx() / 2
  • A core function updates Bins' density and area, and a hotspot.
  • A core function does FFT to compute electrostatic potential and forces and then update to each Bin
  • A core function computes and update WL force using WeightedAverage model
  • Computes the next steps using the ratios of changes of Coordi, Sum Grads

(LOOP_END: BkTrk)

  • Updates parameters for iteration, along with Coefi of hpwl and penalty
  • Calls resizer to assess the worst timing paths, when timing driven mode is enabled
  • Diverge detection on large max_phi_cof value + large design Revert back to original solutions, or break if no way to revert.
  • saves snapshots for routability-driven
  • checks routability using GR when routability driven mode is enabled, revert for bad routability
  • updates DB by marking instances Placed, and set their locations
